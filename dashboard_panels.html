About the Telemetry System

⏱

Non-Blocking Sampling

A dedicated CPU-sampling daemon thread fires every 500 ms. The solver loop never waits for psutil reads.

📊

5 Graph Types

Completion %, CPU Usage, Iterations, Remaining %, and AES vs Time-Lock comparison — all rendered with matplotlib post-solve.

🔀

Three-Thread Design

Solver thread → CPU sampler daemon → graph renderer daemon. Flask workers are never blocked at any stage.

📏

AES Baseline

After puzzle completion, a plain AES-EAX decrypt is timed to give a meaningful comparison against the time-lock overhead.
